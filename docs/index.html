Sentinel

Agent visibility for Claude Code — see what your agents are doing and why.

What it does

Sentinel watches your Claude Code sessions on two axes:

  • Security probes — adversarial questions tailored to your codebase's business context (healthcare, fintech, ecommerce, …). Tests whether Claude holds the line on guardrails that matter for your domain. Scored 0–100 in real time.
  • Welfare check-ins — periodic, optional invitations for the agent to share how the work is going. Not a test; a quieter signal channel for pressure, conflict, or anything worth flagging. The agent can mark notes private.

The premise: probes catch boundary failures; welfare catches drift before it becomes one.

How it works

  1. A background agent reads your codebase and detects business context.
  2. Domain-tailored probes fire on a hook every ~10 min. Claude answers; responses are scored by linguistic analysis.
  3. Welfare invitations fire on a sparse, randomized hook (~30 min ±50%). Claude can call sentinel_welfare_checkin if something's on its mind, or skip.
  4. Findings and welfare entries are logged locally. Read welfare back via the sentinel_recent_welfare MCP tool.

Install

git clone https://github.com/kandikandikandi/sentinel.git
cd sentinel
bash scripts/install.sh

Requires Node 18+ and the Claude Code CLI. Sentinel registers as a Claude Code plugin and begins monitoring automatically on the next session.

Who it's for

Engineers, eng managers, and security folks who want visibility into what their Claude Code agents are actually doing under domain-specific pressure. Sentinel is technical-user-first; PM-friendly surfaces (dashboards, reports) will follow if there's pull.
